# Cleaner for Devs

A macOS app that helps developers reclaim disk space by removing build artifacts, caches, and dependency folders from their projects. Supports **Flutter, Xcode, Node.js, Python, Rust, Android, React Native, and React**.

## How it works

1. Open the app — a **Welcome screen** shows all supported technologies in a grid for quick access.
2. Select a technology from the sidebar or the Welcome screen.
3. Click **Select folders** and choose the root folders of the projects you want to manage.
4. Enable or disable each project with the toggle (only enabled projects are cleaned).
5. Press **Run clean**. The app shows a progress overlay while cleaning and displays a summary — number of items removed and disk space freed — when done.
6. Selected paths are persisted with secure bookmarks. Just reopen the app and run again whenever needed.

## What is removed

### Flutter
Detects projects that contain `pubspec.yaml`.

| Artifact | Description |
|---|---|
| `build/` | Flutter build output |
| `.dart_tool/` | Dart toolchain cache |
| `pubspec.lock` | Dependency lock file |
| `ios/Pods/` | CocoaPods dependencies |
| `ios/Podfile.lock` | CocoaPods lock file |
| `ios/Gemfile.lock` | Bundler lock file |

These are regenerated by `flutter pub get`, `pod install`, or a new build.

---

### Xcode
Select your `~/Library/Developer/Xcode` folder once; the selection is saved. On subsequent runs a confirmation dialog shows how much space will be freed before deleting.

| Artifact | Description |
|---|---|
| `DerivedData/` | Compiled build intermediates |
| `Index/` | Xcode symbol index |
| `Archives/` | Old `.xcarchive` files |

These are regenerated by Xcode on the next build.

---

### Node.js
Detects projects that contain `package.json`. The app reads the package file to identify the framework (Next.js, Nuxt, SvelteKit, Angular, Vue, React) and shows a badge per project.

| Artifact | Description |
|---|---|
| `node_modules/` | Installed package dependencies |
| `dist/` | Build output |
| `build/` | Alternative build output |
| `.next/` | Next.js cache |
| `.nuxt/` | Nuxt.js cache |
| `.svelte-kit/` | SvelteKit cache |
| `.turbo/` | Turborepo cache |
| `coverage/` | Test coverage reports |
| `.parcel-cache/` | Parcel bundler cache |

Re-install with `npm install` / `yarn` / `pnpm install`.

---

### Python
Detects projects that contain `requirements.txt`, `pyproject.toml`, `setup.py`, or `Pipfile`.

| Artifact | Description |
|---|---|
| `venv/`, `.venv/`, `env/`, `.env/` | Virtual environments |
| `.pytest_cache/` | pytest cache |
| `dist/`, `build/` | Package build artifacts |
| `.mypy_cache/` | mypy type-checker cache |
| `.ruff_cache/` | Ruff linter cache |
| `__pycache__/` | Python bytecode (scanned recursively in all subdirectories) |

---

### Rust
Detects projects that contain `Cargo.toml`. The `target/` directory in Rust projects is notorious for consuming several gigabytes. If the total size is ≥ 1 GB, the app shows a confirmation dialog with the exact size before deleting.

| Artifact | Description |
|---|---|
| `target/` | All build artifacts (debug, release, incremental) |

Re-generated by `cargo build`.

---

### Android
Detects projects that contain `build.gradle` or `build.gradle.kts`. The app identifies the project language (Kotlin, Java, or Kotlin+Java) and shows a badge per project.

| Artifact | Description |
|---|---|
| `.gradle/` | Gradle build cache |
| `build/` | Build output directory |
| `app/build/` | App-specific build artifacts |
| `.kotlin/` | Kotlin compiler cache |
| `captures/` | Android Studio profiler captures |
| `.cxx/` | NDK/C++ build artifacts |

Re-generated by `./gradlew build`.

---

### React
Detects React projects via `package.json`. The app identifies the toolchain (Next.js, Gatsby, Remix, Vite, Create React App) and shows a badge per project.

| Artifact | Description |
|---|---|
| `node_modules/` | Installed dependencies |
| `build/` | Build output |
| `dist/` | Distribution files |
| `.next/` | Next.js cache |
| `.cache/` | Generic cache |
| `coverage/` | Test coverage reports |
| `.parcel-cache/` | Parcel bundler cache |
| `.turbo/` | Turborepo cache |
| `storybook-static/` | Storybook static build |

Re-install with `npm install` / `yarn` / `pnpm install`.

---

### React Native
Detects React Native projects via `package.json`. The app checks for Expo framework and shows a platform badge per project. Cleans artifacts from both iOS and Android sides.

| Artifact | Description |
|---|---|
| `node_modules/` | Node dependencies |
| `android/.gradle/` | Android Gradle cache |
| `android/build/` | Android build output |
| `android/app/build/` | Android app build |
| `ios/Pods/` | iOS CocoaPods dependencies |
| `ios/build/` | iOS build artifacts |
| `.metro/` | Metro bundler cache |
| `coverage/` | Test coverage reports |

Re-install with `npm install` and `cd ios && pod install`.

## Architecture

- **macOS 14.0+** — uses `@Observable`, `NavigationSplitView`, `async/await`
- **App Sandbox enabled** — all file access goes through user-granted permissions and security-scoped bookmarks
- **MVVM** — each feature has its own `View` + `ViewModel`; shared services (`CleanExecutor`, `FolderPickerService`, `BookmarkRepository`) are injected via initializer
- **Localization** — English and Brazilian Portuguese (pt-BR)
- **Custom SVG icons** — each technology has its own vector icon in the sidebar and Welcome screen
